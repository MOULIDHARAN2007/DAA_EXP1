Two ways to find a number in a sorted line.

Binary search always cuts the array exactly in half. Interpolation search makes an educated guess about where the target probably sits, the way you'd flip toward the back of a phone book for a name starting with "T." Step through both, side by side, and see exactly where they agree and where they diverge.

Reference

How each one decides where to look

Binary Search

Always probes the exact middle of the remaining range, regardless of the values stored there.

mid = low + floor((high − low) / 2)
  • Best case O(1)
  • Average case O(log n)
  • Worst case O(log n)
  • Assumes about data none — only sorted

Interpolation Search

Estimates the target's position using linear interpolation, like guessing a page in a dictionary.

pos = low + floor((target−arr[low]) × (high−low) / (arr[high]−arr[low]))
  • Best case O(1)
  • Average case O(log log n)
  • Worst case O(n)
  • Assumes about data roughly uniform distribution
